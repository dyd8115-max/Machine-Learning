# 7. 지도학습과 비지도학습

---

## 1. 지도학습 vs 비지도학습 — 패러다임 비교

| 항목 | 지도학습 (Supervised) | 비지도학습 (Unsupervised) |
|------|----------------------|--------------------------|
| 학습 목표 | 입력 $x$와 레이블 $y$의 쌍으로 $P(y \mid x)$ 또는 $f(x)=y$ 추정 | 레이블 없이 데이터 자체의 구조 발견 → $P(x)$ 또는 잠재 구조 추정 |
| 핵심 질문 | "이 데이터가 어느 클래스인가?" / "출력 값이 얼마인가?" | "데이터가 어떻게 그룹화되는가?" / "데이터의 핵심 구조는 무엇인가?" |
| 학습 데이터 | $\{(x_1,y_1), (x_2,y_2), \ldots, (x_n,y_n)\}$ — 레이블 필요 | $\{x_1, x_2, \ldots, x_n\}$ — 레이블 불필요 |
| 대표 알고리즘 | KNN, 로지스틱 회귀, SVM, MLP | K-means, GMM, PCA, Autoencoder |

> **핵심 차이:** 지도학습은 '정답이 있는 학습', 비지도학습은 '정답 없이 패턴을 찾는 학습'이다.

---

## 2. KNN — 직관과 개념

> "새로운 데이터 포인트 $x$가 주어졌을 때, 훈련 데이터 중 **가장 가까운 $K$개의 이웃**을 찾아 **다수결(분류)** 또는 **평균(회귀)**으로 예측한다."

### 핵심 아이디어

**명시적 모델 없음** — 파라미터를 학습하지 않는다 → Instance-based / **Lazy Learning**

**유사성 기반** — 비슷한 입력은 비슷한 출력을 가진다는 가정

**K는 하이퍼파라미터** — K가 작으면 복잡한 경계, K가 크면 부드러운 경계

### 분류 예시

```
K=3 이웃 탐색 → 클래스 A: 2표, 클래스 B: 1표 → 다수결: 클래스 A로 분류
```

---

## 3. KNN — 거리 측도

### 유클리드 거리 (L2)

$$d(x, z) = \sqrt{\sum_i (x_i - z_i)^2}$$

두 점 사이의 **직선 거리**. 연속형 데이터에 가장 많이 사용. 모든 특성의 스케일이 같아야 함 (정규화 필요).

### 맨해튼 거리 (L1)

$$d(x, z) = \sum_i |x_i - z_i|$$

**격자 이동 거리** (택시 거리). 이상치(outlier)에 더 강건. 고차원에서 유클리드보다 안정적.

### 민코우스키 거리 (Lp)

$$d(x, z) = \left(\sum_i |x_i - z_i|^p\right)^{1/p}$$

유클리드($p=2$)와 맨해튼($p=1$)의 **일반화**. $p \to \infty$이면 체비쇼프 거리(최대 성분 차이).

> ⚠️ **스케일 정규화 필수:** 키(cm)와 체중(kg)을 그대로 사용하면 단위 차이로 거리가 왜곡된다. 반드시 정규화(Normalization) 또는 표준화(Standardization)를 먼저 수행해야 한다.

---

## 4. KNN — 분류 알고리즘 단계

| 단계 | 이름 | 내용 |
|------|------|------|
| ① | 훈련 (Train) | 훈련 데이터 $\{(x_1,y_1),\ldots,(x_n,y_n)\}$를 메모리에 저장. 별도의 파라미터 학습 없음 (Lazy Learning) |
| ② | 거리 계산 (Predict) | 새로운 데이터 $x_\text{test}$와 모든 훈련 데이터 간의 거리 $d(x_\text{test}, x_i)$를 계산 |
| ③ | 이웃 선택 | 거리 기준으로 정렬 → 가장 가까운 $K$개의 이웃 $\{x_{i_1}, \ldots, x_{i_K}\}$ 선택 |
| ④ | 예측 (분류) | 이웃의 레이블 다수결 투표 |
| ④' | 예측 (회귀) | 이웃 $K$개의 출력값 평균: $\hat{y} = \dfrac{1}{K}\displaystyle\sum_j y_{i_j}$ |

> **시간복잡도:** 훈련 $O(1)$ — 예측 $O(n \cdot d)$ | **공간복잡도:** $O(n \cdot d)$ → 데이터가 클수록 예측이 느려짐

---

## 5. KNN — 수치 예시

**설정:** 훈련 데이터 5개, 분류 대상 $x_\text{test} = (5, 4)$, $K=3$, 유클리드 거리 사용

| 데이터 | $x_1$ | $x_2$ | 레이블 | $d(x_\text{test}, x_i)$ | 이웃? |
|--------|--------|--------|--------|-------------------------|-------|
| $x_1$ | 3 | 4 | A | $\sqrt{(5-3)^2+(4-4)^2} = 2.00$ | ✓ (1위) |
| $x_2$ | 6 | 2 | B | $\sqrt{(5-6)^2+(4-2)^2} = 2.24$ | ✓ (2위) |
| $x_3$ | 4 | 6 | A | $\sqrt{(5-4)^2+(4-6)^2} = 2.24$ | ✓ (3위) |
| $x_4$ | 8 | 3 | B | $\sqrt{(5-8)^2+(4-3)^2} = 3.16$ | × |
| $x_5$ | 1 | 2 | A | $\sqrt{(5-1)^2+(4-2)^2} = 4.47$ | × |

**다수결:** $K=3$ 이웃 — 클래스 A: 2표, 클래스 B: 1표 → **$\hat{y}$ = 클래스 A**

> **K 값의 영향:** K=1이면 $x_1$(A)만으로 A. K=5이면 A:3표, B:2표 → A. K가 너무 크면 경계가 단순해져 과소적합(underfitting) 발생 가능.

## 6. KNN — K 값의 영향과 Bias-Variance Tradeoff

K가 결정 경계의 복잡도를 제어한다.

| | K = 1 | K = n | 최적 K |
|--|-------|-------|--------|
| 특징 | 가장 가까운 1개만 참조 | 모든 훈련 데이터가 이웃 | 교차 검증으로 선택 |
| 결과 | 훈련 오류 = 0, 결정 경계 매우 복잡 | 항상 다수 클래스로 분류, 결정 경계 단순 | 편향과 분산의 균형점 |
| 문제 | 과적합 (Overfitting) | 과소적합 (Underfitting) | — |
| Bias / Variance | 낮은 편향 / **높은 분산** | **높은 편향** / 낮은 분산 | 균형 |

### K 선택 방법: k-fold 교차검증

```
훈련 데이터를 k개 폴드로 분할
→ 각 폴드를 순서대로 검증 데이터로 사용
→ 검증 오류가 최소인 K를 선택
```

> 홀수 K 권장 (동점 방지). 일반적으로 $K = \sqrt{n}$을 시작점으로 설정.

## 7. KNN — 차원의 저주 (Curse of Dimensionality)

> ⚠️ **KNN의 가장 치명적인 약점 — 차원이 높아질수록 거리 개념이 무의미해진다.**

### 발생하는 문제들

**1. 거리 집중 현상** — 모든 점들이 거의 같은 거리에 위치 → '가장 가까운 이웃' 구분 불가

**2. 노이즈의 증가** — 변수가 늘어날수록 예측과 무관한 노이즈 변수가 늘어날 확률 증가

예시: 환자 데이터로 암 분류 시

| 특성 변수 | 변수 | 예측과의 연관성 |
|-----------|------|----------------|
| 종양 크기 | $x_1$ | 매우 높음 |
| 세포 모양 불규칙도 | $x_2$ | 매우 높음 |
| 환자 혈액형 | $x_3$ | 거의 없음 |
| 환자 생년월일 | $x_4$ | 거의 없음 |
| 병원 방문 요일 | $x_5$ | 거의 없음 |

→ 혈액형이 같은 환자가 종양 크기가 비슷한 환자보다 더 가까운 이웃으로 선택될 수 있음

**3. 계산 비용 증가** — 거리 계산 비용 $O(n \cdot d)$ → $d$가 크면 예측이 매우 느려짐

### 해결 방향

> 특성 선택(Feature Selection) 또는 **PCA로 차원을 줄인 뒤** KNN을 적용하면 성능이 크게 향상된다.

## 8. KNN — 장단점 및 실용적 고려사항

| ✓ 장점 | ✗ 단점 |
|--------|--------|
| 구현이 단순 (핵심 가정 없음) | 예측 시간 $O(n \cdot d)$ — 대규모 데이터에 느림 |
| 훈련이 필요 없음 (Lazy Learning) | 차원의 저주 — 고차원에서 성능 저하 |
| 다중 클래스 분류를 자연스럽게 지원 | 메모리에 전체 훈련 데이터 저장 필요 |
| 비선형 결정 경계를 자동으로 표현 | 스케일에 민감 — 반드시 정규화 필요 |
| 새로운 훈련 데이터 추가가 쉬움 | 특성 중요도를 알 수 없음 (해석 불가) |

### 실용적 체크리스트

| 항목 | 내용 |
|------|------|
| 전처리 | Min-Max 정규화 또는 Z-score 표준화 수행 |
| K 선택 | k-fold 교차검증으로 최적 K 탐색 (홀수 권장) |
| 거리 선택 | 연속형 → 유클리드 / 이산형 → 맨해튼 / 문자열 → 편집 거리 |
| 속도 개선 | KD-Tree 또는 Ball-Tree 자료구조로 이웃 탐색 가속화 |

## 9. K-means — 직관과 개념

> "$n$개의 데이터를 $K$개의 클러스터로 분할하되, 각 데이터는 가장 가까운 클러스터 중심(centroid)에 배정되고, 클러스터 내 분산이 최소가 되도록 중심을 반복 갱신한다."

### 목적함수 (Inertia / WCSS)

$$J = \sum_k \sum_{x_i \in C_k} \|x_i - \mu_k\|^2 \quad \to \quad \underset{C_1,\ldots,C_k,\,\mu_1,\ldots,\mu_k}{\arg\min}\; J$$

$\mu_k$: 클러스터 $k$의 중심 (centroid) | $C_k$: 클러스터 $k$에 속한 데이터 집합

### KNN과의 핵심 차이

| | KNN | K-means |
|--|-----|---------|
| 학습 유형 | **지도학습** | **비지도학습** |
| 레이블 | 필요 | 불필요 |
| 목적 | 기존 레이블 기반 분류 | 새로운 그룹 구조 발견 |

---

## 10. K-means — 알고리즘 단계 (Lloyd's Algorithm)

| 단계 | 이름 | 내용 |
|------|------|------|
| ① | 초기화 (Initialize) | $K$개의 클러스터 중심 $\mu_1, \mu_2, \ldots, \mu_K$를 임의로 선택 (랜덤 샘플링 또는 K-means++ 방식 사용) |
| ② | 할당 단계 (Assignment) | 각 데이터 $x_i$를 가장 가까운 클러스터 중심에 할당: $c_i = \arg\min_k \|x_i - \mu_k\|^2$ |
| ③ | 갱신 단계 (Update) | 각 클러스터의 새로운 중심을 해당 클러스터 데이터의 **평균**으로 갱신: $\mu_k = \dfrac{1}{|C_k|}\displaystyle\sum_{x_i \in C_k} x_i$ |
| ④ | 반복 (Repeat) | ②→③를 클러스터 할당이 변하지 않을 때까지 반복. 수렴 조건: 모든 $c_i$가 이전 반복과 동일할 때 |

> **수렴 보장:** 각 반복에서 목적함수 $J$는 단조 감소 → 반드시 유한 번 반복 후 수렴. (단, **전역 최솟값 보장은 안 됨**)

---

## 11. K-means — 수치 예시

**설정:** 데이터 = $\{1, 2, 8, 9, 10\}$, $K=2$, 초기 중심 $\mu_1=1, \mu_2=9$ (1차원, 유클리드 거리)

### 반복 1 — 할당 단계

| 데이터 | $d(x, \mu_1=1)$ | $d(x, \mu_2=9)$ | 배정 |
|--------|-----------------|-----------------|------|
| $x=1$  | $\|1-1\|=0$     | $\|1-9\|=8$     | $C_1$ |
| $x=2$  | $\|2-1\|=1$     | $\|2-9\|=7$     | $C_1$ |
| $x=8$  | $\|8-1\|=7$     | $\|8-9\|=1$     | $C_2$ |
| $x=9$  | $\|9-1\|=8$     | $\|9-9\|=0$     | $C_2$ |
| $x=10$ | $\|10-1\|=9$    | $\|10-9\|=1$    | $C_2$ |

### 반복 1 — 갱신 단계

$$\text{새 } \mu_1 = \frac{1+2}{2} = 1.5 \qquad \text{새 } \mu_2 = \frac{8+9+10}{3} = 9.0$$

### 반복 2 — 할당 & 갱신

$C_1=\{1,2\}$, 배정 동일 → $\mu_1=1.5$ (불변)  
$C_2=\{8,9,10\}$, 배정 동일 → $\mu_2=9.0$ (불변) → **수렴!**

### 최종 결과

| 클러스터 | 데이터 | 중심 |
|----------|--------|------|
| $C_1$ | $\{1, 2\}$ | 1.5 |
| $C_2$ | $\{8, 9, 10\}$ | 9.0 |

반복 횟수: 2회 | $J = 0.5 + 0.5 + 1 + 0 + 1 = 3.0$

---

## 12. K-means — 수렴 보장 및 한계

### 수렴 증명 (직관)

**① 할당 단계** — 각 $x_i$를 가장 가까운 중심에 배정 → 거리 합 $J$가 감소하거나 같아짐 ($J_\text{new} \leq J_\text{old}$)

**② 갱신 단계** — 클러스터 중심을 평균으로 갱신 → 평균은 제곱 오차를 최소화하는 점 → $J$가 감소하거나 같아짐

**③ 유한성** — 클러스터 할당의 경우의 수는 유한 ($K^n$개) → 단조 감소 + 유한 → **반드시 수렴**

### 주의: 전역 최솟값 비보장

초기 중심값에 따라 서로 다른 지역 최솟값(local minimum)에 수렴할 수 있다.

> **해결책:** 여러 번 랜덤 초기화 후 가장 작은 $J$ 값의 결과 선택 (`n_init` 파라미터)

## 13. K-means — 초기화 문제와 K-means++

### 랜덤 초기화의 문제

초기 중심이 같은 클러스터 내에 여러 개 배치되면 → 빈 클러스터 또는 나쁜 지역 최솟값 발생 → 실제 클러스터 구조를 제대로 못 찾음

### K-means++ 초기화 절차

| 단계 | 내용 |
|------|------|
| 1 | 첫 번째 중심 $\mu_1$을 데이터에서 균일 랜덤 선택 |
| 2 | 각 데이터 $x_i$에 대해 이미 선택된 중심들까지의 최소 거리² 계산: $D(x_i)^2 = \min_k \|x_i - \mu_k\|^2$ |
| 3 | $D(x_i)^2$에 비례한 확률로 다음 중심 선택: $P(x_i \text{ 선택}) \propto D(x_i)^2$ |
| 4 | $K$개의 중심이 선택될 때까지 2~3 반복 |
| 5 | 이후 표준 K-means 알고리즘 실행 |

> K-means++는 초기화만 바꿨을 뿐이지만 수렴 속도와 최종 품질을 크게 개선한다. **scikit-learn 기본값으로 사용됨.**

## 14. K-means — K 개수 선택 (Elbow Method)

### 핵심 아이디어

$K$를 늘릴수록 $J$ (inertia)는 감소한다. $K=n$이면 $J=0$ (자기 자신이 중심) → **단순히 $J$가 작은 $K$를 고르면 안 됨**.

### Elbow 기준

$J$ 감소율이 급격히 줄어드는 **'elbow' 지점** → 추가 클러스터의 이득이 크지 않은 $K$ 선택

### 절차

```
K = 1, 2, 3, ..., 10에 대해 K-means 실행
각 K에서의 J 값을 기록
J vs K 그래프에서 꺾이는 지점(elbow) 선택
```

> ⚠️ 항상 명확한 elbow가 보이지 않을 수 있다. 이 경우 **실루엣 계수(Silhouette Score)** 등 다른 지표를 병행한다.

## 15. K-means — 한계 및 주의사항

| 한계 | 설명 | 대안 |
|------|------|------|
| ① K를 사전에 지정 | 클러스터 개수를 미리 알아야 함 | Elbow method, 실루엣 점수로 탐색 |
| ② 구형(원형) 클러스터 가정 | 유클리드 거리 기반 → 타원형/복잡한 모양 포착 어려움 | DBSCAN, GMM |
| ③ 이상치(Outlier)에 민감 | 중심 = 평균이므로 이상치가 중심을 크게 왜곡 | K-medoids (중앙값 기반) |
| ④ 스케일에 민감 | 특성 스케일이 다르면 특정 특성이 지배 | 표준화 (Z-score) 필수 |
| ⑤ 이진/범주형 데이터 부적합 | 유클리드 거리가 의미 없음 | K-modes, Gower 거리 |

> K-means는 단순하고 빠르지만 이러한 한계를 이해하고 데이터 특성에 맞는 알고리즘을 선택해야 한다.

## 16. KNN vs K-means — 체계적 비교

| 비교 항목 | KNN | K-means |
|-----------|-----|---------|
| 학습 유형 | 지도학습 (Supervised) | 비지도학습 (Unsupervised) |
| 레이블 필요 | 필요 ✓ | 불필요 ✗ |
| K의 의미 | 이웃 개수 (탐색 범위) | 클러스터 개수 (그룹 수) |
| 훈련 단계 | 없음 (Lazy) | 반복 최적화 |
| 예측 방식 | 다수결 투표 / 평균 | 가장 가까운 centroid 배정 |
| 시간복잡도(예측) | $O(n \cdot d)$ | $O(K \cdot n \cdot d \cdot t)$ ($t$=반복 횟수) |
| 스케일 민감성 | 매우 민감 (정규화 필수) | 매우 민감 (정규화 필수) |
| 결과 해석 | 분류 / 회귀 예측값 | 클러스터 레이블 (군집 구조) |
| 주요 응용 | 분류, 회귀 | 고객 세분화, 이상 탐지, 압축 |

> **핵심:** KNN은 '이미 알려진 레이블로 새 데이터를 분류'하고, K-means는 '레이블 없이 데이터의 숨겨진 그룹 구조를 발견'한다.

## 17. 전체 ML 파이프라인에서의 위치

```
데이터 수집          전처리/정규화       탐색적 분석         특성 선택/PCA       모델 학습           평가/검증
(Raw Data)    →   (Preprocessing)  →  (EDA + K-means)  →  (차원 축소)      →  (KNN/SVM/MLP)  →  (Confusion Matrix)
                                             ↑                                      ↑
                                         K-means 활용                          KNN 활용
```

### K-means 주요 활용 사례

고객 세분화 (CRM): 구매 패턴으로 고객군 분류, 이미지 압축: 픽셀 색상 군집화로 색상 수 감소, 문서 군집화: 유사 주제 문서 자동 그룹화, 이상 탐지: 클러스터에서 멀리 떨어진 데이터 탐지

### KNN 주요 활용 사례

추천 시스템: 유사 사용자 기반 콘텐츠 추천, 의료 진단: 유사 환자 데이터 기반 질병 예측, 이미지 인식: 픽셀 특성 유사도 기반 분류, 이상 거래 탐지: 정상 패턴과 거리 기반 탐지

## 요약

| 키워드 | 핵심 내용 |
|--------|-----------|
| 지도 vs 비지도 | 지도학습은 레이블로 $f(x)=y$를 학습. 비지도학습은 레이블 없이 $P(x)$의 구조를 발견 |
| KNN 핵심 | $K$개 이웃의 다수결(분류) 또는 평균(회귀). 모델 학습 없는 Lazy Learning. 거리 정규화 필수 |
| KNN 주의 | K 선택은 교차검증. 고차원에서 차원의 저주 발생 → PCA로 선처리 필요 |
| K-means 핵심 | $K$개 클러스터, 반복 할당-갱신. 목적함수 $J = \sum\|x_i - \mu_k\|^2$. 수렴은 보장, 전역최솟값은 미보장 |
| K-means 주의 | K-means++ 초기화 권장. Elbow/Silhouette으로 K 선택. 구형 클러스터 가정 한계 |
